## 1. ブランドカラー仕様

### 1.1 プライマリカラー

| カラー名 | Hex Code | RGB | 用途 |
|---------|----------|-----|------|
| **Dark Gray** | `#333333` | RGB(51, 51, 51) | タイトル、見出し、テキスト |
| **Medium Gray** | `#666666` | RGB(102, 102, 102) | セカンダリテキスト、フッター |

### 1.2 セカンダリカラー

| カラー名 | Hex Code | RGB | 用途 |
|---------|----------|-----|------|
| **Background White** | `#FFFFFF` | RGB(255, 255, 255) | 背景 |
| **Light Gray** | `#F5F5F5` | RGB(245, 245, 245) | セクション背景、カード背景 |
| **Text Gray** | `#333333` | RGB(51, 51, 51) | 本文テキスト |
| **Caption Gray** | `#666666` | RGB(102, 102, 102) | キャプション、補足テキスト |

### 1.3 カラー使用ルール

#### ✅ 推奨される使い方
- タイトル・見出しには必ず **Dark Gray** (`#333333`)
- 本文は **Dark Gray** (`#333333`)
- フッターやサブ要素には **Medium Gray** (`#666666`)
- 背景は原則 **White** (`#FFFFFF`)
- **モノクロデザインを厳守** - ホワイト、ブラック、グレーのみ使用

#### ❌ 禁止事項
- カラフルな色を使用しない（赤、青、緑、黄色など）
- カラーコードを勝手に変更しない
- グラデーションは使用しない（ブランドトンマナに合わない）
- FABRIC TOKYOはミニマル×モノクロが基本

## 2. タイポグラフィ仕様

### 2.1 フォントファミリー

#### 日本語フォント

| 優先順位 | フォント名 | 用途 | フォールバック |
|---------|-----------|------|--------------|
| 1 | **游ゴシック Medium** | タイトル・見出し | Yu Gothic Medium → Hiragino Sans → Noto Sans JP |
| 2 | **游ゴシック** | 本文・キャプション | Yu Gothic → Hiragino Sans → Noto Sans JP |

#### 英数字フォント

| 優先順位 | フォント名 | 用途 |
|---------|-----------|------|
| 1 | **DIN** | 数字表示（オプション） |
| 2 | **Yu Gothic** | 一般的な英数字 |

### 2.2 フォントサイズ階層

| 要素 | サイズ（pt） | 太さ | 行間 | 用途 |
|------|------------|------|------|------|
| **大見出し（H1）** | 36pt | Bold | 1.4 | タイトルスライドのメインタイトル |
| **中見出し（H2）** | 28pt | Bold | 1.4 | コンテンツスライドのタイトル |
| **小見出し（H3）** | 24pt | Medium | 1.5 | セクション見出し |
| **本文** | 20pt | Regular | 1.6 | 通常のコンテンツテキスト |
| **キャプション** | 14pt | Regular | 1.5 | 注釈、フッター |

### 2.3 行間・文字間隔

#### 行間（Line Height）

```yaml
タイトル: 1.4倍
本文: 1.6倍
キャプション: 1.5倍
```

**重要**: 行間は必ず守ること。これが改行割れ防止の最重要要素。

#### 文字間隔（Letter Spacing）

```yaml
タイトル: 0 (標準)
本文: 0 (標準)
キャプション: 0 (標準)
```

FABRIC TOKYOのデザインは詰め過ぎず、広げ過ぎず、標準の文字間隔を維持。

## 3. レイアウト原則

### 3.1 スライドサイズ

```yaml
幅: 10インチ (25.4cm)
高さ: 7.5インチ (19.05cm)
比率: 4:3
```

### 3.2 マージン・余白

#### 基本マージン

```yaml
上: 0.8インチ (2.03cm)
下: 0.8インチ (2.03cm)
左: 0.8インチ (2.03cm)
右: 0.8インチ (2.03cm)
```

#### セーフティゾーン

重要なテキストや要素は、**マージンから更に0.2インチ内側**に配置。

```
実質的な安全エリア:
  上下左右: 1.0インチ (2.54cm)
```

### 3.3 グリッドシステム

#### 8ポイントグリッド

すべての要素の配置は、8の倍数（8pt, 16pt, 24pt, 32pt...）でスナップさせる。

```
例:
  要素間のスペース: 16pt, 24pt, 32pt
  テキストボックスの高さ: 48pt, 64pt, 80pt
```

### 3.4 ホワイトスペース（余白）の原則

#### FABRIC TOKYOの余白哲学

> 「余白は贅沢である」

- 要素を詰め込まない
- 最小限の情報で最大限の印象を
- 呼吸するデザイン

#### 具体的な余白ルール

```yaml
セクション間: 最低32pt以上
見出しと本文: 24pt
段落間: 16pt
箇条書き項目間: 12pt
```

## 5. デザイン原則

### 5.1 ミニマリズム

#### コンセプト
> 「Less is More」- 情報を削ぎ落とし、本質だけを残す

#### 実装ルール
- 1スライドに1メッセージ
- 箇条書きは最大5項目まで
- 装飾的な要素は最小限に
- グラフィック要素は意味のあるものだけ

### 5.2 高級感の演出

#### 要素
1. **余白**: 贅沢に使う
2. **カラー**: 落ち着いたトーン
3. **フォント**: クリーンでモダン
4. **画像**: 高品質のみ使用

#### 避けるべきこと
- 過度な装飾
- 派手なアニメーション
- 多色使い
- 低解像度の画像

### 5.3 統一感

#### 一貫性のルール
- すべてのスライドで同じフォントファミリー
- カラーパレットを厳守
- レイアウトの規則性を保つ
- マージン・余白を統一

## 6. 改行割れ・文字つぶれ防止ルール

### 6.1 改行割れ防止

#### 原因
- テキストボックスの高さ不足
- 行間が狭すぎる
- 自動改行が無効

#### 対策

```yaml
必須設定:
  lineSpacing: 行間を1.4〜1.6倍に設定
  wrap: true (自動折り返しを有効化)
  breakLine: true (改行を保持)
  height: 十分な高さを確保（最低でもフォントサイズの3倍以上）
```

#### 高さの計算式

```
テキストボックスの最低高さ =
  (フォントサイズ × 行間) × 予想行数 + 余裕分(0.5インチ)

例: 20ptフォント、行間1.6、3行の場合
  = (20 × 1.6) × 3 + 0.5
  = 96pt + 0.5インチ
  ≈ 1.8インチ
```

### 6.2 文字つぶれ防止

#### 原因
- 自動縮小機能（AutoFit）が有効
- フォントサイズが小さすぎる
- テキスト量が多すぎる

#### 対策

```yaml
必須設定:
  autoFit: false (自動縮小を無効化)
  fontSize: 最低14pt以上（本文は20pt推奨）

推奨:
  1スライドの文字数: 最大200文字
  箇条書き: 1項目あたり最大40文字
```

### 6.3 日本語特有の問題対策

#### 問題
- 英語フォントで日本語が表示されると読みにくい
- 文字間隔が不自然になる

#### 対策

```yaml
フォント指定:
  fontFace: "游ゴシック" (日本語フォントを明示)

フォールバック:
  "游ゴシック" → "Yu Gothic" → "Hiragino Sans" → "Noto Sans JP"
```

## 9. トラブルシューティング

### 問題: タイトルが改行される

**原因**: テキストボックスの幅が不足

**解決策**:
1. テキストボックスの幅を8.4インチに設定
2. タイトルを簡潔にする（最大30文字推奨）
3. 2行タイトルの場合、高さを2.0インチ以上に設定

### 問題: 本文が読みにくい

**原因**: 行間が狭い、または文字サイズが小さい

**解決策**:
1. フォントサイズを20ptに設定
2. 行間を1.6に設定
3. 段落間スペースを16pt確保

### 問題: デザインがFABRIC TOKYOっぽくない

**原因**: カラーまたはレイアウトがブランドガイドラインから逸脱

**解決策**:
1. カラーコードを厳密にチェック
2. 余白を増やす
3. 情報を削ぎ落とす（ミニマル化）
4. フォントを游ゴシックに統一
